# Route Changes

## Route Renaming

### Student → Parent

All student routes renamed to parent:

| Old Route  | New Route | Purpose                   |
| ---------- | --------- | ------------------------- |
| `/student` | `/parent` | Browse tutors anonymously |

**Impact:**

- Rename `src/app/student/` → `src/app/parent/`
- Update all `Link` components pointing to `/student`
- Update navigation in Header component
- Update landing page links

## New Routes

### Authentication Routes

| Route           | Type   | Purpose                              |
| --------------- | ------ | ------------------------------------ |
| `/auth/signin`  | Public | NextAuth sign-in page (Google OAuth) |
| `/auth/signout` | Public | Sign out confirmation                |
| `/auth/error`   | Public | Authentication error page            |

**Note:** These are auto-generated by NextAuth.js via `/api/auth/[...nextauth]/route.ts`

### Dashboard Routes

| Route              | Type      | Purpose                  |
| ------------------ | --------- | ------------------------ |
| `/tutor/dashboard` | Protected | Tutor profile management |

**Protection:** Requires authentication via middleware

## Route Structure

```text
/
├── / (landing page)
│   ├── "I'm a Parent" → /parent
│   └── "I'm a Tutor" → /auth/signin?callbackUrl=/tutor/dashboard
│
├── /parent (public)
│   └── Browse all tutors
│
├── /tutors/[id] (public)
│   └── View single tutor profile
│
├── /auth/* (public, NextAuth)
│   ├── /auth/signin
│   ├── /auth/signout
│   └── /auth/error
│
└── /tutor/dashboard (protected)
    └── Create/edit tutor profile
```

## Removed Routes

| Old Route | Action  | Replacement                    |
| --------- | ------- | ------------------------------ |
| `/tutor`  | Removed | `/tutor/dashboard` (protected) |

**Note:** Old `/tutor` page contained public registration form. This is now moved to the protected dashboard.

## Middleware Configuration

```typescript
// src/middleware.ts
export { default } from "next-auth/middleware";

export const config = {
  matcher: [
    "/tutor/dashboard/:path*",
    // Add more protected routes here in future
  ],
};
```

**Behavior:**

- Unauthenticated users redirected to `/auth/signin`
- After sign-in, redirected back to original destination
- Session checked on every request to protected routes

## API Routes (No Changes)

| Route             | Method | Purpose                     |
| ----------------- | ------ | --------------------------- |
| `/api/tutors`     | GET    | Fetch all tutors            |
| `/api/tutor/[id]` | GET    | Fetch single tutor          |
| `/api/submit`     | POST   | Create/update tutor profile |

**Note:** `/api/submit` will be updated to:

- Require authentication
- Link tutor to logged-in user via `userId`
- Support both create and update operations
